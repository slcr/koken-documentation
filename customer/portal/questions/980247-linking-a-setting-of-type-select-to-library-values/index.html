Todd Dominey Apr 22, 2013 02:41PM EDT Koken developer

So that’s why you were asking about loading albums by id. Gotcha. :)

I just tried this with an internal build and it works (and will work in 0.8):

https://gist.github.com/tdominey/11401e5865fb5fb722b3

(I’m loading the smallest preset image in that example for speed/dev purposes.)

Couple of things to note here. One, Koken’s API limits requests to 100 items, so you’ll only be able to display 100 or fewer items from each album. Pagination usually handles situations like these, but that works when the whole page is loading a single content source. There isn’t a way to paginate within each koken:load that appears in a template, if that makes sense. But if you’re not loading that many items it wouldn’t affect you.

This could also — depending on the # of albums in the set — be a heavy load on the server and browser. Depends really on the size of images you’re loading, etc. Just something to keep in mind.

So I think the best plan for now would be to wait for 0.8 then give my code example above a try to see if it meets the need.
